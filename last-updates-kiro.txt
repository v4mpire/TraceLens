> # TraceLens Project Context Report

## Project Overview
TraceLens is a self-hosted observability platform that transforms web application debugging by focusing on

causality rather than metrics. It's a Runtime Truth Engine with AI integration that builds real-time 
dependency graphs from browser and server runtime signals.

- **Type**: Turborepo monorepo with 7 packages + 1 web app
- **Current State**: 100% complete (13/13 checkpoints)
- **NPM Packages**: 4 published packages (browser-sdk, server-sdk, shared, mcp-server)
- **Version**: All packages at v1.0.0

## Architecture

### Monorepo Structure
TraceLens/
├── packages/
│   ├── browser-sdk/          # Client-side performance monitoring
│   ├── server-sdk/           # Backend tracing and dependency SDK
│   ├── ingestion-service/    # Event processing and data normalization
│   ├── analysis-engine/      # Causal graph construction and analysis
│   ├── dashboard/            # Next.js web interface
│   ├── security-scanner/     # CVE mapping and vulnerability analysis
│   ├── shared/               # Common types and utilities
│   └── mcp-server/           # Model Context Protocol server (PUBLISHED)
├── apps/web/                 # Main dashboard application
├── tools/                    # Shared configurations (ESLint, TypeScript)
└── docker/                   # Production deployment configs


### Key Architectural Patterns
- **Event-driven architecture**: Asynchronous data processing
- **Causal graph construction**: Deterministic dependency analysis
- **MCP integration**: AI-queryable interface via Model Context Protocol
- **Microservices design**: Independent scaling and deployment
